## `lib/` — client utilities

| Path | Purpose |
|---|---|
| `lib/supabase.js` | Supabase client singleton. Reads `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`. Configured to persist sessions in localStorage. |
| `lib/auth.js` | `consumeLoginFragmentIfPresent()` — on app load, reads `#login=<base64>` from the URL fragment, decodes `email:password`, signs in with Supabase, replaces the hash with `#/app`. Also `signOut()`. |

## `context/` — React context providers

| Path | Purpose |
|---|---|
| `context/AuthProvider.jsx` | `<AuthProvider>` wraps the app. Exposes `useAuth()` returning `{ session, profile, loading }`. Calls `consumeLoginFragmentIfPresent` once on mount, then watches Supabase auth state changes. Fetches the user's profile row (display name, avatar, `is_admin`) when a session exists. |

## `scripts/` — one-off / admin scripts

| Path | Purpose |
|---|---|
| `scripts/seed_users.mjs` | Creates Supabase Auth users + matching `public.users` rows for the participants listed in the `USERS` array. Prints a personal login link for each. ## `supabase/migrations/` — SQL migrations

Run these in the Supabase SQL editor in order.

| Path | Purpose |
|---|---|
| `supabase/migrations/0001_init.sql` | Initial schema: `users`, `initial_photos`, `final_photos`, `check_ins`, `points` tables; the `leaderboard` public view; the `is_admin()` helper function; RLS policies for all data tables. |
| `supabase/migrations/0002_grants.sql` | Postgres GRANTs for `anon`, `authenticated`, `service_role` on all `public` tables, sequences, and functions. Includes `alter default privileges` so future tables inherit the right grants. Needed because RLS bypass only works if grants exist. |
| `supabase/migrations/0003_storage_policies.sql` | RLS policies on `storage.objects` for the `photos` bucket. Authenticated users can read any photo (group sharing); only the owner (or an admin) can write to `<user_id>/...` paths. |

## Path conventions in Supabase Storage (`photos` bucket)

```
<user_id>/initial/front.jpg
<user_id>/initial/side.jpg
<user_id>/initial/back.jpg
<user_id>/final/front.jpg
<user_id>/final/side.jpg
<user_id>/final/back.jpg
<user_id>/checkin/<YYYY-MM-DD>/scale.jpg     # week_start date as folder
```

The first path segment must be the auth user's UUID. RLS enforces this.
